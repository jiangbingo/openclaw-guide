# 故障排除

本章收集了安装和使用 OpenClaw 过程中常见的问题及其解决方向。

::: tip 提示
以下内容为通用性问题的解决思路。针对具体环境和特定错误的详细解决方案，需要根据实际情况分析。
:::

## 安装阶段问题

### Q: Homebrew 安装失败

**可能原因**：
- 网络连接问题
- macOS 版本过旧
- 权限问题

**通用解决方向**：
1. 检查网络连接
2. 尝试使用国内镜像源
3. 确保有足够的磁盘空间

### Q: 依赖安装报错

**可能原因**：
- 系统库版本冲突
- 缺少开发工具
- 路径配置问题

**通用解决方向**：
1. 更新 Xcode Command Line Tools
2. 检查环境变量设置
3. 尝试单独安装问题依赖

### Q: 构建过程失败

**常见错误类型**：
- 编译器错误
- 链接错误
- 资源不足

**通用解决方向**：
1. 查看详细错误日志
2. 检查是否有足够的内存
3. 确认所有依赖已正确安装

## 运行阶段问题

### Q: 启动失败

**可能原因**：
- 配置文件错误
- 端口被占用
- 权限不足

### Q: 运行缓慢

**可能原因**：
- 资源占用过高
- 缓存未启用
- 配置不当

### Q: 功能异常

**可能原因**：
- 环境变量问题
- 配置参数错误
- 版本兼容性问题

## 系统相关问题

### Q: macOS 版本过低警告

Old Mac 可能运行的是较旧的 macOS 版本，某些功能可能受限。

### Q: 安全设置阻止运行

macOS 的安全机制可能会阻止未签名应用运行。

**通用解决方向**：
- 在系统偏好设置中允许运行
- 使用命令行移除隔离属性

## 日志查看

当遇到问题时，查看日志是排查的重要手段：

```bash
# 查看主日志
cat ~/.openclaw/logs/main.log

# 查看错误日志
cat ~/.openclaw/logs/error.log
```
